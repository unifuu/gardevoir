## はじめに

転職活動は、それ自体がフルタイムの仕事のようなものです。企業研究、応募書類の作成、面接対策、そして終わりのない進捗管理...。やるべきタスクは山積みです。

そこで私は、Anthropic の新技術 **「Claude Code」 のエージェントスキル機能**を活用し、**就職活動のプロセスをエンジニアリング**しました。

これは単なるタスク管理ではありません。**「AI エージェントに就活のスキルを実装し、調査から対策までを自律的に実行させる」** という、新しいアプローチの実践記録です。

## 🎯 解決した課題

### 😫 従来の問題点（Before）

- **情報のサイロ化**: Excel、スプレッドシート、メモアプリなどに情報が散乱。
- **手動作業が多い**: 企業HPを巡回し、技術スタックを調べ、Obsidianにまとめる...という繰り返しの作業。
- **進捗が見えづらい**: 全体像の把握が困難。

### 🥰 この仕組みで実現したこと（After）

- ✅ **Single Source of Truth**: 全情報を Markdown 化し、Obsidian で一元管理。
- ✅ **ワークフローの自動化**: コマンド一発で、深掘りされた企業研究と面接対策が生成される。
- ✅ **可視化と戦略立案**: AI が全データを横断分析し、次のアクションを提案。

## 🛠 技術スタック

- **Claude Code**: ファイルシステムへのアクセス権を持つ、次世代 AI CLI ツール。
- **Claude Sonnet**: 推論能力とコーディングに特化した大規模言語モデル。
- **Web Search & Fetch API**: リアルタイム情報の収集源。
- **Markdown/Obsidian**: ローカル完結型の構造化ドキュメント管理。

### なぜ Claude Code なのか?

従来の ChatGPT などとの最大の違いは、**ファイルの読み書きができる**こと:

```bash
# ファイルの作成・更新が可能
$ claude "株式会社〇〇について調査"
→ companies/株式会社〇〇.md が自動生成

# 複数ファイルを横断分析
$ claude "レポート作成"
→ 全企業ファイルをスキャンして統計レポート生成
```

## 📦 4つのコアスキル

### 1. 企業研究スキル

**使い方**: 企業名を入力するだけ

```bash
$ claude "株式会社〇〇"
```

**AI が自動実行**:

1. Web 検索で最新情報を収集
2. 以下の項目を自動分析:

- 📋 基本情報(設立年、資本金、従業員数等)
- 💻 技術スタック(使用言語、インフラ、開発環境)
- 🎯 採用要件(求めるスキル、経験年数)
- 🌍 外国人雇用状況(外国籍の社員の比率、社内公用語)
- ⚠️ SES判定(自社開発 vs 客先常駐の比率)
- 👥 社員インタビュー(最低3件、実際の社員の声)
- 💬 口コミ・評判(OpenWorkスコア、退職検討理由、入社後のギャップ)

3. 自分のプロフィールと照合して適合性評価

- マッチ度: ★★★★☆
- 強み: [自分の経験と合致する点]
- 懸念点: [ギャップがある点]

**生成される内容の例**:

```markdown
---
company: 株式会社〇〇
date: 2026-01-16
status: 検討中
match_score: 4
---

- **設立**: 2018年 (ベンチャー/シリーズC)
- **規模**: 従業員数 150名 / 資本金 5億円

- **Backend**: **Go, Python** (Django)
- **Frontend**: React, TypeScript, Next.js
- **Infra**: AWS (ECS, Fargate), Terraform, Datadog

## 🎯 採用要件

- **必須**: Web開発経験3年以上、静的型付け言語での実装経験
- **歓迎**: マイクロサービスアーキテクチャの設計経験

## 🌍 外国人雇用状況

- **比率**: **約15%** (エンジニアチームは20%超)
- **言語**: 社内公用語は日本語だが、**仕様書は英語併記**が標準。
- **実績**: 直近1年で台湾・韓国・ベトナムから計8名採用。

## ⚠️ SES判定: 非SES (完全自社開発)

- **判定結果**: 自社プロダクト100%のため、**客先常駐は無し**。
- **勤務**: フルリモート可（週1出社推奨）。

## 👥 社員インタビュー (要約分析)

AIが抽出した3名のリアルな声：

1.  **バックエンド (3年目)**: "Goへのリプレイスを若手主導で完遂できた。裁量は非常に大きい。"
2.  **テックリード (5年目)**: "外国籍メンバーも多く、コードレビューは英語で行われることもある。"
3.  **採用人事**: "技術力だけでなく、プロダクト志向を強く求めている。"

## 💬 口コミ・評判 (OpenWork/Vorkers)

- **スコア**: ⭐ **3.8 / 5.0** (業界平均以上)
- **Positive**: 技術レベルの高さ、フラットな組織、モダンな環境。
- **Negative/退職理由**: 急成長による組織の歪み。「プレイングマネージャーへの負担が大きい」との声あり。

## ✅ 適合性評価: ★★★★☆ (4/5)

- **強み**: モダンな技術スタックで最新技術に触れられる。
- **懸念**: 急成長中のため、プロセスやドキュメントが未整備な部分がある。

## 🔗 参考リンク

- **公式サイト**: https://example.jp
- **採用ページ**: https://example.jp/careers
- **Wantedly**: https://www.wantedly.com/companies/example
```

### 2. 求人分析スキル

**使い方**: 求人URLを貼り付けるだけ

```bash
$ claude "https://example-job-site.jp/job/12345"
```

**AI が自動実行**:

1. 求人ページ全文を解析
2. **全ての応募要件を一つずつ分解**
3. 各要件に対して面接回答例を生成(STAR法)
4. 想定質問と対策を提案

**生成される面接対策の例**:

```markdown
# [会社名] - 求人分析・面接対策

## 必須条件 1: Go言語でのバックエンド開発経験 3年以上

**解釈**:
設計能力とマイクロサービスアーキテクチャの理解が求められる

**あなたの該当経験**:

- プロジェクト: 決済システム開発
- 期間: 2021年〜2024年(3年)
- 成果: 処理速度30%向上、可用性99.99%達成

**面接での回答例**:
状況: 前職で決済サービスのバックエンドを担当
課題: レガシーなモノリシックシステムをマイクロサービス化
行動:

- Go言語でgRPCベースのマイクロサービスを設計
- Kubernetesでオートスケーリング実装
  結果:
- 処理速度30%向上
- 可用性99.99%達成

**想定質問**:

- "Go言語を選んだ理由は?"
- "マイクロサービス化で苦労した点は?"

## 📝 逆質問リスト

1. "チーム内のコードレビュー文化について教えてください"
2. "技術的な成長機会について具体例を教えてください"

## 📚 準備チェックリスト

- [ ] Go言語プロジェクトのポートフォリオ整理
- [ ] Kubernetes実装例準備
- [ ] システム設計の説明準備
```

### 3. ステータス管理スキル

```bash

```

ファイルのYAML frontmatterが自動更新され、履歴が記録されます。

## 🎨 実際のワークフロー

私の就活は、ターミナルの中で完結します。

1. Discover: `$ claude "株式会社〇〇"` → 深い企業分析が即座に完了。
2. Analyze: `$ claude "求人URL"` → 自分の経歴に最適化された「台本」が完成。
3. Update: `$ claude "〇〇社 応募完了"` → ステータス管理も会話のみ。
4. Review: `$ claude "レポート作成"` → 客観的なデータで戦略を修正。

## 💡 従来との違い

| 項目             | 従来             | この仕組み                       |
| ---------------- | ---------------- | -------------------------------- |
| **情報管理**     | チャット画面のみ | ファイルとして永続保存           |
| **コンテキスト** | 毎回説明が必要   | 全企業データを横断分析           |
| **情報収集**     | 手動でWeb検索    | 企業名入力で自動収集             |
| **分析深度**     | 表面的           | 社員インタビュー、口コミまで収集 |

## 🔧 技術的な工夫

### スキルベースアーキテクチャ

Claude Code の「スキル」機能を活用:

```markdown
# .claude/skills/company_research.md

## Trigger

research, 調査, 会社, 企業, 株式会社

## Instructions

1. Web検索で企業情報を収集
2. 技術スタックを分析
3. 社員インタビューを検索
4. マッチ度評価
5. Markdownファイルを生成
```

トリガーワードを検出すると自動的にこのスキルが起動します。

## 📝 まとめ

この仕組みは、単なる「便利ツール」ではありません。**AI エージェントと協働する新しい働き方**の実証です。

```
従来: 人間 → 情報収集 → 分析 → 判断 → 行動

現在: 人間(戦略・判断) ⇄ AI(情報収集・分析) → 協働で最適な行動
```

**AI時代のエンジニアとしての働き方**を体現しました。最新技術を学ぶだけでなく、実際の課題に適用し、価値を創出する。それが、これからの時代に求められる力だと信じています。
